In an earlier Mythos Madness post, I used MATH-500 and Humanity's Last Exam to make a point about models and harnesses. Useful benchmarks. Insurance-correlated, perhaps. But not insurance.

For insurance, we need to test submissions, claim files, underwriting guidelines, loss triangles, policy language, and the ability to justify a decision in a way that will stand up to a market conduct exam.

So at InsureThing, we started building InsureBench: an insurance-specific evaluation system for testing both AI models and the harnesses around them on tasks that look like the work carriers, brokers, and agents need to get done every day.

That distinction matters. We are not testing chat. We are testing work, done with real tools.

You would not test an actuary on arithmetic without giving them a calculator. LLMs also need data, context, and tools to perform at the level they are really capable of. A file reader matters. A Python environment matters. A retrieval layer, workflow, or agentic CLI may matter even more.

InsureBench is designed to test the whole setup.

The preview version scores model-plus-harness combinations on real insurance work: classifying messy Workers' Comp risks, reading loss triangles, auditing claims against policy terms, spotting coverage gaps, and handling unclear facts. The questions are written and reviewed by insurance experts. Where answers can be checked by code, they are. Where judgment is required, we grade against expert rubrics.

The early results look intuitively right, which is a good start. It is not the same as declaring victory. We are adding questions, tools, task types, and more demanding examples daily.

The most interesting early result is also the most practical: a smaller model in a serious agentic harness can beat larger models running in simpler API setups. The whole system matters more than the model alone. An inexpensive model with the right harness can be much cheaper to run than an expensive model producing the same useful result.

The next step is to go deeper by task type. Not just which model is best overall, but which model and harness are good enough for a specific job.

Yes, a frontier model running at high reasoning will usually score better overall than a small model you can run behind a firewall. But if the smaller model is fast, secure, reliable, and paired with a harness that lets it perform well on one specific task, the overall benchmark may not matter.

Some tasks will justify the expensive model. Many will not. InsureBench is designed to help identify which tasks belong with which model, which harness, and which deployment constraints.

These early results are previews based on a smaller question set, currently broken into broad categories like claims, underwriting, and actuarial work. The numbers will move as we add questions, increase complexity, repeat runs, and improve the harness lanes. As the task bank grows, the results will get more specific and more useful.
